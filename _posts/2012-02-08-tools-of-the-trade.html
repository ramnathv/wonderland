--- layout: post title: Tools of the Trade ---

I’ve been working with GovHub for going on three months now, and at this point I think I’ve finally gotten my development environment refined.

GovHub is built on Symfony 1.4 with the Doctrine ORM, and deployed on AWS S3. Since we’re such a small team, I have my hands in pretty much every aspect of the development, and knowing which tools to use and when to use them is key.

#1: Github for Mac

Switching our version control from SVN to Git was like being born-again. Definitely one of those things where you can never imagine going back. Github does a great job hosting our repo, and their bug tracker “Issues” is fantastic as well.

Github for Mac is a beautiful application, but its abstraction layer is almost too thick. For everyday use it has a simple interface & is easy to understand, but for things like merging branches and stashing changes I have no choice but to turn to the command line.

#2: Textmate

Textmate is a cult favorite if there ever was one. (Apparently a screenshot of the long-awaited Textmate 2 in Alpha was the most tweeted photo on instagram for a day.) Switching from a full-blown IDE (Netbeans) to Textmate took a bit of configuration and adjustment. 

Tip #1: Install ProjectPlus, which gives you an IDE-like sidebar for viewing your project’s files. It also gives you VCS support, if you’re into that thing. Personally, if I want to see which files I’ve modified, I just pop open Github for Mac.

Tip #2: Use Command+T and Command+Shift+T. The former searches for files in your project, the latter searches for definitions within the currently-viewed file. What used to be a right click to “find declaration” in Netbeans is now a Command+T to search for the class, and possibly a Command+Shift+T to find the definition inside of it. Sure, it’s a little less convenient, but every time Netbeans would freeze up and then prompt me with “Netbeans was unresponsive for 6008ms, would you like to send a bug report?” I wanted to kill myself.

Tip #3: Install AckMate for searching. Like a true UNIX noob, I didn’t know about ack before using AckMate, so let me tell you: this shit is fucking fast. I can scan our entire 40MB codebase for a regular expression in ~10 seconds, and I don’t even know what the hell I’m doing most of the time.

#3: Sequel Pro

Alright, so I guess I’m working backwards, since I haven’t even gotten to MAMP yet. But Sequel Pro is amazing. Keeping our database sync’ed across our Production, Stage, and local servers would normally require that someone actually knows what they’re doing. Not with Sequel Pro: I can export our 360MB Production database and import it onto my local machine in less than 10 minutes. Brilliant, and a lot easier for now than creating a custom data set for local development.
